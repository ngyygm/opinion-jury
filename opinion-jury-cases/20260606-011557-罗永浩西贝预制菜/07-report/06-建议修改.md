# 6. 建议修改 (Recommended Revisions)

本节基于四组模拟辩论（24位角色、144轮发言）中所有攻击向量的完整回溯分析，逐条给出原文文本修改、术语纠正、配套措施和战略建议。每条建议均标注优先级（MUST_FIX / RECOMMENDED / OPTIONAL），并引用具体模拟攻击证据形成完整的证据链：建议 --> 所中和的攻击 --> 攻击者的私人动机 --> 原始数据证据。

**原文：**

**所中和的攻击向量：**

## 6.7 方法论说明

本节所有建议均遵循以下证据链要求：

1. **定位攻击：** 从144轮 say.public.json 中定位针对原文具体措辞的攻击言论。
2. **还原动机：** 从对应的 turn.private.json 中提取攻击者的 inner_monologue 和 private_goal，确认攻击是真诚关切还是策略性操作。
3. **验证分类：** 对照 claim-gate-packet.v001.json 中对应 claim 的 classification（VERIFIED / UNVERIFIED / CONTRADICTED / OPINION_PRESENTED_AS_FACT），确认原文在该点上的证据强度。
4. **评估影响：** 对照 full-replay-analysis.json 中的 modification_priority、misrepresentation_targets 和 collateral_victims，评估该攻击向量在真实舆论中的放大潜力。
5. **提出修改：** 确保修改后的文本同时满足三个条件：(a) 消除或削弱该攻击向量；(b) 保留原文的核心论点和价值立场；(c) 不引入新的、更易被攻击的弱点。

每条建议的优先级判定标准：
- **MUST_FIX：** 对应的 claim 在 claim-gate 中被标记为 CONTRADICTED 或 UNVERIFIED（置信度 LOW），且该攻击在模拟中产生了显著的舆论转向效果。
- **RECOMMENDED：** 对应的 claim 在 claim-gate 中被标记为 OPINION_PRESENTED_AS_FACT 或 UNVERIFIED（置信度 MEDIUM），且模拟中至少有两个独立角色提出了相关质疑。
- **OPTIONAL：** 对应的 claim 证据强度尚可，但在模拟中仅被个别角色提及，且修改可能引入新的表述成本。
